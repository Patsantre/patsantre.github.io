Модель хранения личной базы знаний

Как устроено хранение в персональной «LLM-вики»: слои от горячего дистиллята до холодного архива, принцип единого источника истины и правила жизненного цикла данных. Обобщённая методическая схема — без конкретных систем и данных.
hot — дистиллят warm — сырьё structured — индекс corpus — поиск cold — bulk

Слои хранения

🔥 hotдистиллят
База знаний (вики)
Человекочитаемый дистиллят, разбитый по разделам жизни. Источник истины. Держится в бюджете токенов LLM, чтобы целиком влезать в контекст модели.
канон≤ ~200k токеновверсионируется (git)
🌡 warmсырьё
Слой источников
Иммутабельные исходники, из которых выведен дистиллят: транскрипты, выгрузки, импорт из приложений. Только дописывать — провенанс фактов.
append-onlygit
🧊 structuredиндекс
Структурный слой
Реляционный/табличный индекс для запросов, где это удобнее markdown. Должен быть либо производным от дистиллята, либо иметь чёткое разграничение ролей — иначе двойная истина и дрейф.
queryable
🔎 corpusпоиск
Документный корпус
Полнотекстовый/векторный поиск по большим наборам документов, которые не место в дистилляте. Отдельный движок-поиск.
full-text / vector
❄️ coldbulk · архив
Холодное хранилище
Тяжёлые файлы, медиа, бэкапы — всё, чему не место в git-репозитории базы. Дистиллят ссылается на них по предсказуемому пути и контрольной сумме (манифест).
вне gitссылка по md5

Принцип источника истины

Один класс — один канонУ каждого класса данных ровно один источник истины. Любой дубль — потенциальный дрейф, его нужно либо устранить, либо явно подчинить (одно генерится из другого).
Hot читает, не дублируетДистиллят не копирует тяжёлое и структурное, а ссылается на cold/structured. Так бюджет остаётся маленьким, а факты — едиными.
OKF: один концепт — один файлКаждая страница = один концепт с типом во frontmatter. Это даёт машинную проверяемость и экспорт.

Жизненный цикл данных

① Промоция warm → hot

Новое из входящих/сырья дистиллируется в базу + ставится указатель на источник.
держит бюджет дистиллята маленьким

② Демоция hot → cold

триггер: бюджет переполняется или деталь переросла выжимку
  • длинная деталь → слой источников (без потерь)
  • завершённое → архив
  • тяжёлое → холодное + ссылка-манифест

③ Ретенция warm

порог по объёму/возрасту
  • самый быстрорастущий поток — автозахват
  • старое → в холодное (сжатие)
  • держит рабочее дерево лёгким
